## 便签纸张纹理改造：SVG feTurbulence → 图片噪点纹理

### 问题
原始 SVG `feTurbulence` 滤镜实现纸张纹理，但：
- 运算在 CPU（导致滚动/交互掉帧）
- 各浏览器渲染效果不一致
- 无法实现每张便签不同的纹理偏移

### 方案
用静态 PNG 噪点图片 + CSS `::before`/`::after` 伪元素代替

### 实施细节
1. **生成 high-contrast 噪点 PNG**（Python/PIL）
   - 灰度范围 10-250（原图仅 158-177，几乎不可见）
   - 平均不透明度 79%（原图仅 22%）
   - 尺寸 300×300，241 种颜色
2. **文件部署**：`public/noisy.png` → 构建后 `dist/noisy.png`，引用路径 `/noisy.png`
3. **代码修改**：`src/components/todo/TodoPanel.vue`
   - ✅ 移除 SVG `filter id="paperNoise"` 定义（含 feTurbulence/feColorMatrix/feComposite/feBlend）
   - ✅ `.sticky-note`：移除 `filter: url(#paperNoise)`，新增 `::before` 伪元素做纹理
   - ✅ `.note-label-tape`：移除 `filter: url(#paperNoise)`，新增 `::after` 伪元素做纹理
   - ✅ 纹理样式使用 CSS 变量 `--tex-blend` / `--tex-opacity`，JS 响应式注入
   - ✅ 每张便签随机 `background-position`（通过 `getNoteStyle` 注入 `--bg-pos` CSS 变量）
   - ✅ 浅色模式：`multiply + 0.15`，深色模式：`overlay + 0.3`
4. **性能优势**：GPU 纹理叠加，零 CPU 计算

## Bug 修复：拖拽唯一待办后出现虚拟便签

### 问题
把 TodoPanel 中最后一条待办拖到日历后，API 返回空列表 → `loadTodos()` 回退加载 VIRTUAL_TODOS（5条示例便签）

### 根因
`CalendarView.onDayDrop` 删除后端待办 → `emit('today-created')` → Home.vue `todoRefreshKey++` → TodoPanel `watch(refreshKey)` 触发 `loadTodos()` → API 空 → 错误地加载了虚拟便签

### 修复
1. `loadSync()` 完成后主动 `emit('today-created')` 触发 TodoPanel 刷新，解决首次加载钉子待办不显示的竞态
2. 钉子待办(source===1)拖到同一天日期 → 直接 return 跳过，不做无效操作

## 修复：跨组件拖拽视觉效果"删没"的问题

### 问题
拖拽日历 ↔ 待办面板时，被拖拽的项视觉效果上被删除，除非刷新页面才正常。

### 根因
SortableJS 的 `onEnd`（源容器）在 `onAdd`（目标容器）之前触发。两个组件的 `onEnd` 都执行了 `window.__dragPayload = null`，导致目标 `onAdd` 读取 payload 时已经是 `null`，跨组件拖拽逻辑被静默吞掉。

### 修复（两个组件）

**CalendarView.vue `onEnd`**：
- 检测到跨组件（`evt.to` 没有 `data-date` 属性）→ 立即从本地数据移除防止视觉闪烁，return 不做任何操作
- `__dragPayload` 不清空（留给目标 `onAdd` 处理）
- 日历内部拖拽（同列/跨列）逻辑不变

**TodoPanel.vue `onEnd`**：
- 同列排序逻辑不变
- 跨组件：非钉住待办（source!=1）立即从本地数据移除；钉住待办（source==1）保留，由事件监听器处理
- 跨组件不清理 `__dragPayload`

**Payload 清理兜底**：
所有 `onAdd` 中提前 return 的路径都加上了 `window.__dragPayload = null`，防止残留

### 核心原则
`__dragPayload` 生命周期由目标容器的 `onAdd` 全权管理，源容器的 `onEnd` 绝不触碰。

## 修复：钉住待办拖到当日日期栏直接消失

### 问题
钉住待办（source=1）从 TodoPanel 拖到日历当日栏目后，DOM 消失，只刷新网页才出现。

### 根因
- TodoPanel `onEnd`（source=1）：保留数据（正确 — 钉住待办不应该被删除）
- Calendar `onAdd`（同一天）：return 不做任何操作
- 但 SortableJS 已经把 DOM 从 TodoPanel 移走了，数据没变 → Vue 不重渲染 → DOM 永久缺失

### 修复
Calendar `onAdd` 中，钉住待办拖回同一天时，派发 `cal-pinned-restore` 事件：
```js
window.dispatchEvent(new CustomEvent('cal-pinned-restore'))
```
TodoPanel 监听该事件，强制 Vue 重渲染（`todos.value = [...todos.value]`），
Vue 对比虚拟 DOM 发现缺少元素后重新插入。

### 关键规则
`__dragPayload` 生命周期由目标容器的 `onAdd` 全权管理，源容器的 `onEnd` 绝不触碰。

## 数据库表合并：calendar_todos → todos（统一模型）

### 背景
原来两张表（todos + calendar_todos）存储待办，通过 source/source_id 字段关联，导致：
- 跨组件拖拽数据同步逻辑极其复杂（delete+create 模式）
- 勾选/删除需要二次同步，易出竞争态
- 钉住待办（source=1）需要特殊处理

### 方案
移除 `CalendarTodo` 模型，`Todo` 模型新增 `date` 字段担当区分角色：
- `date IS NULL` = 右侧待办面板
- `date` 有值 = 摸鱼日历待办
- `reminder_at`/`reminded`/`updated_at` 字段从 calendar_todos 合并过来

### 迁移
1. 迁移脚本 `backend/migrate_merge_tables.py` 处理：
   - 向 todos 表添加 date/reminder_at/reminded/updated_at 列
   - 将原有 calendar_todos 数据合并进 todos
   - 重建表结构，删除 source/source_id/due_date 列
   - 删除 calendar_todos 表
2. `calendar_todo.py` 重命名为 `.bak`
3. `source`/`source_id` ALTER TABLE 从 `_migrate_columns()` 中移除

### 改动文件
- **backend/models/todo.py** — 统一 Todo 模型
- **backend/models/__init__.py** — 移除 CalendarTodo 导入
- **backend/routes/calendar.py** — 改用 Todo 模型
- **backend/routes/todo.py** — 查询包含 date IS NULL + date=today 并集
- **backend/app.py** — 移除废弃的 source/source_id 列添加
- **src/api/calendar.js** — 移除未使用的 syncToday 导出
- **src/components/todo/TodoPanel.vue** — source===1 → date!==null，移除同步逻辑，简化拖拽

### 注意事项
- `date` 字段使用 `%Y-%m-%d` 字符串格式（SQLite 无原生 DATE 类型，但 ORM 映射为 Python date）
- 日历 API 端点 `/api/calendar/*` 保留，向后兼容

## 双面板数据联动（2026-05-15 16:11）

### 问题
表合并后 CalendarView 不再 emit `today-created`，TodoPanel → CalendarView 方向也完全没有刷新机制，两个面板数据不同步。

### 方案
Home.vue 作为协调器，通过 refreshKey prop + emit 事件双向联动：
- CalendarView 加 `refreshKey` prop + watcher，emit `todo-changed`
- TodoPanel 加 emit `todo-changed`
- Home.vue 监听双方事件，分别递增对方的 refreshKey
- 所有 CRUD 操作（新增/修改/勾选/删除/跨天拖拽/跨组件拖拽）均触发通知

### 踩坑：当日待办拖拽重复 Bug

**问题**：当 `date=today` 的待办从日历拖到右侧面板时，在右侧面板中重复出现（列表中原有一份 date=today 的记录，`unshift(res.data)` 又加了一份新的）。

**根因**：`/api/todos` 返回 `date IS NULL OR date=today` 的并集，今日日历待办同时在两个面板中显示。拖拽时 `onAdd` 直接 `unshift` 新数据，没考虑已存在的情况。

**修复**：`onAdd` 中先 `find` 本地列表是否已有该 todo ID，有则 `Object.assign` 原地更新，无则 `unshift` 插入。
